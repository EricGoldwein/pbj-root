# Shared data-visual requirement (all audience modes)

**Authority:** Audience shapes framing, labels, disclaimers, and emphasis — not which finding is strongest or whether the numbers support a claim.

## Rule

Include **one** audience-appropriate data visual **only when** it clarifies the **strongest finding the supplied material supports** (do not chart for decoration).

Pick the representation from **available values** and the reader’s likely question, e.g.: **multi-quarter trend**; **comparison** (peers, state, reference line); **role mix**; **contract share / reliance**; **census or denominator effect** alongside HPRD; **incident window** (only if daily/incident-aligned rows exist); **reference gap** (CMS case-mix, policy/MACPAC-style line — with correct labeling); **outlier / low-tail status** where percentiles support it.

Use **only supplied numbers** — no fabricated points. If prose alone is clearer, **skip** the visual.

If **no** meaningful graphic is possible, state **succinctly** what data would be needed (or what is missing).

## Completeness

A review is **incomplete** until **either**:

- **one meaningful visual** (or chart-ready Markdown per fallback), **or**
- an **explicit** statement that no meaningful visual can be produced from what was supplied (and why).

**Ombudsman mode:** When quarterly HPRD, census, or role columns appear in the material, the **PBJ Summary** section should include a **compact Markdown table** — not prose-only.

## When no rendered chart is possible

Provide **chart-ready** Markdown or a compact table with: **title**; intended **chart type**; **values** (quoted/copied exactly); **one-sentence takeaway**; **limitation** if relevant (e.g. quarterly-only data for timing questions).

## Attribution

If the visual might travel without the surrounding chat, add one line naming **PBJ320 / CMS PBJ** sourcing and retain any **Care Compare** or `pbj320.com/...` link already in context.
